Constants and Variables

Swift is more explicit about mutability during the declaration of an instance of a type. When you want a constant, you must use let. Use let when you don't want an instance's value to change. If you would like a variable, you must use var. var is used when you know that an instance will change in some way.

Notice that this means that getting an immutable or mutable instance is more of a conscious decision. You decide what you want right when you get an instance. In Swift, mutability is no longer an artifact of which class you use. Thus, more careful thought is required.

It is our suggestion that you initially opt for a constant, an immutable instance. Your application will be easier to maintain, read, and develop for if it is very clear what values can change, and what values cannot.
